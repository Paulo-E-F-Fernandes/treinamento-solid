# Pasta Docs (Documentação)

É o coração do conhecimento do seu projeto.

Enquanto o `README.md` principal dá uma visão geral, a pasta `docs` contém o aprofundamento técnico ou manual de uso.

**O que colocar aqui:**
* **Guias de Instalação:** Detalhes passo a passo.
* **Manuais de Referência:** Explicação de cada funcionalidade.
* **Tutoriais:** Conteúdo educativo para novos usuários.

**Dica de Navegação:** Se a documentação for longa, divida em subpastas numeradas (ex: `01-basico`, `02-avancado`) para
manter a ordem de leitura.

```markdown
├── docs/                   # Documentação detalhada do projeto
│   ├── 01-guia-usuario/    # Subpasta temática numerada
│   │   ├── instalacao.md
│   │   └── configuracao.md
│   ├── 02-tecnico/
│   │   ├── arquitetura.md
│   │   └── api-reference.md
│   └── faq.md
```
